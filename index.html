GIT系统基本命令行以及操作步骤

在线操作

1.生成SSH密钥

ssh-keygen -t rsa -C “邮箱地址”#密钥会在C盘User下的.ssh文件夹

ssh -T git@github.com#测试链接是否成功

  • 具体步骤参见：https://www.cnblogs.com/superjt/p/5977719.html或

​ https://www.cnblogs.com/sdcs/p/8270029.html>

2.设置默认的用户名与邮箱

git config –global user.name “用户名”

git config –global user.email “邮箱”

git config –list#查看配置列表

3.添加线仓库与本地仓库链接

git remote add origin http://……#注意这里的origin是在线库地址的指代名称

git remote#查看当前配置的远程仓库

git remote -v#查看已经链接的在线库地址与指代名称

本地操作(加粗指令为常用指令)

1.将本地文件夹变为Git可管理的库并上传到Respostry

git init #初始化，在本地文件见内建立.git文件夹，这个文件夹用于缓存

git add .#添加当前文件夹内容到.git文件夹下，做好上传准备，这是每次上传的必备步骤

git add xxx.xxx#添加指定文件到,git文件夹下

git commit -m “注释内容”#添加注释内容，每一次push前必须进行此步，不然无法上传

git push -u origin master#将.git文件夹内容push到origin所代表的仓库中的master分支中

git push origin master#第一次加了-u后之后就不用加了

git push origin master -f#强制push

2.将指定url地址下的内容克隆到本地

git clone git@github.com….

3.从远程仓库下载新的改动

git pull origin master#这里的origin依旧代表了仓库地址

git reset –hard , git pull#忽略本地修改，直接将远程库文件拉下来

4.创建新分支并推到云端

git checkout -b xxx#创建并切换到新分支xxx

git push origin xxx#推送xxx到云端

其他指令

mkdir xxxx#创建文件夹xxxx

cd xxxx/#进入到文件夹xxxx

pwd#显示当前目录

rm xxx#删除xxx文件

git status#查看当前库里的情况

git remote -v#查看远程仓库情况

git remote rm xxx#删除远程仓库xxx

git branch#查看当前本地所有分支

git checkout xxx#切换分支

git branch -r#查看远端分支

git branch -a#查看所有分支

git branch xxx#查看远程分支

git branch -d xxx#删除xxx分支

git merge xxx#合并分支到master

git remote show origin#查看远程分支与本地分支的关系

git stash,git stash,git stash pop#三连备份本地修改，正常pull,然后将本地修改应用到当前工作区

git diff xxx#查看xxx文件的那些内容被修改过了

git log#查看以往版本

git reset –hard HEAD^#滚回上个版本

git reset –hard xxx#滚到某个版本

git reset –hard FETCH_HEAD,git pull#冲掉本地修改，直接将云端内容pull下来

进阶指令可参考：https://www.jianshu.com/p/ac33f0295629c
