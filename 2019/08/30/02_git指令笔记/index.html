GIT系统基本指令以及配置指令笔记

GIt的设计让使用者觉得自己比想象中的笨——Andrew Morton

在线配置指令

1.生成SSH密钥

ssh-keygen -t rsa -C “邮箱地址” #密钥会在C盘User下的.ssh文件夹

ssh -T git@github.com#测试链接是否成功

具体步骤参见：

https://www.cnblogs.com/superjt/p/5977719.html

https://www.cnblogs.com/sdcs/p/8270029.html

2.设置默认的用户名与邮箱（可以理解成注册账户）

git config –global user.name “用户名”

git config –global user.email “邮箱”

git config –list#查看配置列表

3.添加在线仓库与本地仓库链接

git remote add origin http://……#注意这里的origin是在线库地址的指代名称

git remote#查看当前配置的远程仓库

git remote -v#查看已经链接的在线库地址与指代名称

本地操作(加粗指令为常用指令)

1.将本地文件夹变为Git可管理的库并上传到Respostry

git init #初始化，在项目文件夹内建立.git文件夹，这个文件夹用于缓存，git系统通过追踪这个文件夹里的文件来管理项目

git add .#将工作区所谓文件添加到缓存区(.git文件夹)，让git系统开始跟踪文件

git add xxx.xxx#添加指定文件到,git文件夹下

git commit -m “注释内容”#添加注释内容，并提交

git commit –amend -m xxx#修改上次提交的内容

git push -u origin master#将.git文件夹内容上传到云端仓库“origin”所代表的仓库中的master分支中

git push origin master#第一次加了-u后之后就不用加了

git push origin master -f#强制push

2.将指定url地址下的内容克隆到本地

git clone git@github.com….

3.从远程仓库下载新的改动

git pull origin master#这里的origin依旧代表了仓库地址

git reset –hard , git pull#忽略本地修改，直接将远程库文件拉下来

4.创建新分支并推到云端

git checkout -b xxx#创建并切换到新分支xxx

git push origin xxx#推送xxx到云端

回滚指令专辑

reset指令一共可分为三类：

1.git reset [–mixed] HEAD（等同于git reset HEAD）# 移动HEAD的指向，并覆盖回暂存区

git reset HEAD #撤回最后一次commit，让上个提交版本覆盖回暂存区

git reset HEAD~ #撤回最后一次commit，让上个提交版本覆盖回暂存区

git reset HEAD~~#撤回倒数第二次commit，让上上个提交版本覆盖回暂存区

git reset xxx #撤回xxx文件最后一次commit，让其上个版本覆盖回暂存区

git reset xxx #这样可将xxx版本（未来版本）文件覆盖回暂存区，实现“往前滚”

2.git reset –soft HEAD #仅仅移动HEAD的指向，不动另外两个区

git reset –soft HEAD~ #撤销一次错误commit，移动HEAD将其指向上一个快照（仅仅在储存区动作）

3.git reset –hard HEAD#不仅移动HEAD指向，还将文件覆盖回工作区

git reset –hard HEAD^ #撤销上一次commit，让上个版本覆盖回工作区以及暂存区（危）

git reset –hard id#撤销到某个指定xxx版本（配合log使用），覆盖回工作区以及暂存区（危）

git reset –hard id #这样可将xxx版本（未来版本）文件覆盖回暂存区与工作区，实现“往前滚”

git reset –hard FETCH_HEAD,git pull#冲掉本地修改，直接将云端内容pull下来

git reset –hard HEAD~#移动HEAD，将其指向上一个快照，将其覆盖到暂存区以及工作区

4.git checkout#使用暂存区的老文件覆盖掉工作区的新文件（危）

git checkout – xxx #使用暂存区的老文件xxx，覆盖回工作区（危）

删除指令

git rm xxx#删除xxx文件（从工作区以及暂存区删除），如果想从储存区移除，那么就要用reset soft 指令

git rm –cached xxx#仅删除暂存区文件

git rm -f xxx#从工作区与暂存区暴力删除xxx文件，不管二者是否一致

其他指令

1.初始N连击

mkdir xxxx#创建文件夹xxxx

cd xxxx/#进入到文件夹xxxx

pwd#显示当前目录

rm xxx #删除xxx文件

git init #初始化，在项目文件夹内建立.git文件夹，这个文件夹用于缓存，git系统通过追踪这个文件夹里的文件来管理项目

2.文件比较

git diff #比较暂存区以及工作区的文件

git diff xxx1 xxx2#比较两个快照内容

git diff id #比较储存区快照与工作区快照的文件内容

git diff HEAD#比较当前指向的快照与储存区的文件内容

git diff xxx #查看xxx文件的那些内容被修改过了

git diff – cached #比较暂存区与储存区快照

git diff – cached id #比较id文件在暂存区与储存区的快照

3.其他常用指令

git status#查看当前库里的情况，返回分支情况、工作区文件情况（工作区的文件是不受跟踪的，而缓存区与储存区的文件相反）

git log #查看历史提交版本id（只能看到当前HEAD指向目的地之前的版本）

git mv old_file_name new_file_name #同时在三个区域修改文件名称

git remote -v#查看远程仓库情况

git remote rm xxx#删除远程仓库xxx

git branch#查看当前本地所有分支

git checkout xxx#切换分支

git branch xxx #创建分支xxx

git checkout xxx #切换分支到xxx

git branch -r#查看远端分支

git branch -a#查看所有分支

git branch xxx#查看远程分支

git branch -d xxx#删除xxx分支

git merge xxx#合并分支到master

git remote show origin#查看远程分支与本地分支的关系

git stash,git stash,git stash pop#三连备份本地修改，正常pull,然后将本地修改应用到当前工作区
